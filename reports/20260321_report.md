# NewsStock AI 작업 리포트 — 2026.03.21~22

## 작업 요약
랜딩 페이지 전면 구축, 인증 시스템 구현, 대시보드 5개 탭 UI 완성, Supabase DB 연동, 포트폴리오 기능 설계

---

## 1. 랜딩 페이지 구축

### 스플래시 화면
- "NewsStock" 로고 1.2초 표시 후 페이드아웃 → 로그인 화면으로 전환
- Framer Motion 애니메이션 적용

### 로그인/회원가입 화면
- 이메일 + 비밀번호 기반 인증
- 이메일 `@` 형식 유효성 검사
- 회원가입: 이름, 이메일, 비밀번호, 비밀번호 확인
- 회원가입 ↔ 로그인 전환 (페이드 애니메이션)
- Google / Kakao 소셜 로그인 버튼 (UI만, 실제 연동 미완)

### 관리자 계정
- 이메일 칸에 "0" 입력 → 인증 없이 즉시 로그인 (admin@newsstock.com 자동 생성)

## 3. 인증 시스템

### 백엔드 API
- `POST /api/auth/email` — 이메일 회원가입/로그인
- Supabase `users` 테이블 연동 (email, password, name, role, plan 컬럼)
- 로그인 성공 시 쿠키에 유저 정보 저장 (30일 유지)

### Supabase 연동
- `@supabase/supabase-js` 클라이언트 설정
- RLS 정책 설정 완료 (anon key로 CRUD 허용)
- `.env.local`에 API URL + anon key 설정

## 4. 대시보드 (5개 탭)

### 레이아웃
- 왼쪽 사이드바 (데스크톱) / 햄버거 메뉴 (모바일)
- 유저 프로필 + 로그아웃 버튼
- 페이지 전환 시 페이드 애니메이션
- 비로그인 시 랜딩 페이지로 리다이렉트

### 뉴스 피드 (/feed)
- 인사말 + 요약 카드 3개 (오늘의 뉴스, AI 분석, 포트폴리오)
- 뉴스 목록 (카테고리 필터: 전체/정책/기업/글로벌)
- 상승/하락/중립 태그
- 목 데이터 6건
- **향후**: 포트폴리오 종목 관련 뉴스 자동 필터링 구현 예정

### AI 분석 (/analyze)
- URL 입력 → 분석하기 버튼
- 4단계 분석 진행 애니메이션 (기사 파싱 → AI 분석 → 종목 매칭 → 리포트 생성)
- 분석 결과: 핵심 인사이트, 숨겨진 의도, 파장 분석(단기/중기/장기), 수혜 종목, 손해 종목
- 신뢰도 바, PDF 내보내기 버튼
- 목 데이터 (반도체 정책 뉴스 시나리오)

### 포트폴리오 (/portfolio)
- 1단계: 스크린샷으로 연동하기 (이미지 업로드 → AI 분석)
- 2단계: 직접 입력하기 (종목명, 코드, 수량, 매입가, 시장)
- 포트폴리오 목록 표시 + 종목 삭제
- 스크린샷 분석 API (`POST /api/stocks/screenshot`) — Claude Vision API 연동 준비

### 리포트 (/reports)
- 분석 히스토리 목록
- 필터 (전체/상승/하락/중립)
- 각 리포트에 PDF 내보내기 버튼
- 목 데이터 4건

### 설정 (/settings)
- 프로필 (이름, 이메일)
- 요금제 (Free/Premium)
- 알림 설정 (뉴스 알림, 분석 완료 알림, 포트폴리오 변동 알림)

## 5. API 엔드포인트

| 경로 | 메서드 | 설명 |
|---|---|---|
| `/api/auth/email` | POST | 이메일 회원가입/로그인 |
| `/api/auth/phone` | POST | 전화번호 인증 (레거시, 사용 안 함) |
| `/api/stocks/screenshot` | POST | 스크린샷 AI 분석 → 종목 추출 |
| `/api/stocks/kis` | POST | 한국투자증권 API 보유종목 조회 |
