========================================
NewsStock AI 개발 대화 내역
날짜: 2026.03.21 ~ 2026.03.22
========================================

[1] 기술 스택 설명
- 유저가 기술 스택 재설명 요청
- 프론트엔드/백엔드/DB 기술 스택 설명

[2] 백엔드 기술 스택 상세 설명
- Next.js 16, NextAuth.js, Anthropic SDK, Stripe, RSS Parser, 한국투자증권 API, Finnhub API, Zod, bcryptjs 각각 상세 설명

[3] 공부 순서 안내
- HTML/CSS → JavaScript → TypeScript → React → Next.js → NextAuth 순서 안내
- 유저: "바이브코딩 해놓고 코드보고 공부할게"

[4] 개발 서버 시작
- npm install → npm run dev → localhost:3000 실행

[5] 랜딩 페이지 구축
- 요구사항: 웹 → 안드로이드/iOS 변환 고려, 라이트/다크 모드, 페이드 전환, 고딕체
- NewsStock 로고 + 회원가입 (전화번호 인증 방식)
- 구현: React + Framer Motion + Tailwind CSS
- 색상: 흰색+진한초록 / 검정+진한초록

[6] 다크/라이트 모드 자동 전환
- prefers-color-scheme 감지 스크립트 추가
- 시스템 설정 변경 시 실시간 반영

[7] 관리자 계정
- 010-2852-7928 번호로 인증번호 111111 하드코딩
- 이후 인증번호 없이 바로 로그인으로 변경
- 이후 이메일 방식으로 전면 변경

[8] 스플래시 화면
- "NewsStock" 로고만 먼저 표시 → 페이드아웃 → 로그인 화면
- 시간: 2초 → 1.5초 → 1.2초로 조정

[9] Supabase 데이터베이스 연동
- API URL + anon key로 Supabase JS 클라이언트 연결
- users 테이블 생성 (SQL Editor에서 실행)
- RLS 정책 설정

[10] 인증 방식 논의
- SMS 무료 발송 방법 없음 확인
- 이메일 + 소셜 로그인(Google/Kakao)으로 전환 결정
- NextAuth vs Supabase Auth vs 쿠키 직접 관리 비교
- 쿠키 방식으로 일단 구현 (빠르게 만들고 나중에 전환)

[11] 이메일 로그인 구현
- 이메일/비밀번호 회원가입, 로그인 API 구축
- Google/Kakao 소셜 로그인 버튼 UI 추가
- 관리자: "0" 입력 시 즉시 로그인

[12] 홈 화면 (대시보드) 구현
- 라우팅/리다이렉트 개념 설명
- 대시보드 레이아웃: 사이드바 + 모바일 햄버거 메뉴
- 5개 탭: 뉴스피드, AI분석, 포트폴리오, 리포트, 설정
- 각 탭 기본 페이지 생성

[13] 증권사 연동 논의
- 한국투자증권: REST API, 바로 가능
- 키움증권: Windows OCX, 중계 서버 필요
- 토스/카카오/삼성: API 없음
- 증권사 MTS 사용자 수 순위 조사 (미래에셋 1위 360만, 키움 2위 343만)

[14] 포트폴리오 페이지 설계
- 처음: 증권사 연동(한투 API) + 스크린샷 분석 + 직접 입력
- 증권사 아이디/비밀번호 로그인 불가 설명 (보안 규정)
- 최종: 스크린샷 AI 분석 + 직접 입력 (증권사 연동 제거)

[15] 사업계획서 분석
- 예비창업패키지, 창업중심대학 사업계획서 2개 분석
- 주 기능: 뉴스 AI 분석, AI 수혜/손해 종목 도출, 포트폴리오 연관 뉴스
- 부가 기능: 포트폴리오 관리, PDF 내보내기, 분석 히스토리, 전문가 연결, 프리미엄 구독

[16] 각 페이지 목 데이터 구현
- AI 분석: URL 입력 → 4단계 애니메이션 → 수혜/손해 종목 결과
- 뉴스 피드: 뉴스 6건 + 카테고리 필터 + 상승/하락 태그
- 리포트: 분석 히스토리 4건 + 필터 + PDF 버튼

[17] 향후 계획 메모
- 뉴스 피드: 포트폴리오 종목 관련 뉴스 자동 필터링 (구현 예정)
- Claude API 실제 연동은 나중에
