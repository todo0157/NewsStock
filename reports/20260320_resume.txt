Claude: 기술 스택을 카테고리별로 정리.

----------------------------------------

[3] 유저: PostCSS, Prettier, ESLint는 왜 쓰는 거야?

Claude:
- PostCSS: Tailwind CSS를 브라우저용 CSS로 변환하는 도구 (필수)
- Prettier: 코드 스타일(따옴표, 들여쓰기 등) 자동 통일 (맞춤법 교정기)
- ESLint: 버그 될 수 있는 코드 검출 (문법 검사기)

----------------------------------------

[4] 유저: 백엔드/프론트엔드/DB로 나눠서 설명 + Supabase 무료 끝남, 직접 DB 관리하고 싶음

Claude: 기술 스택 3분류로 재정리.
DB 관리 대안 제시:
1. 로컬 PostgreSQL 설치 (개발용 추천)
2. Docker로 실행
3. 클라우드 DB (Neon, Railway, Render)
Supabase URL 기억: https://hbocridxxmqlioigzsal.supabase.co

----------------------------------------

[5] 유저: 로컬 PostgreSQL로 관리하면 배포할 때는?

Claude: 로컬은 개발용, 배포할 때는 클라우드 DB(Neon 등)로 전환.
DATABASE_URL 환경변수만 바꾸면 코드 변경 없이 전환 가능.

----------------------------------------

[6] 유저: DB 배포한다는 게 뭔 말이야? 데이터가 다 공개되는 거 아니야?

Claude: DB 배포 = 테이블 구조만 클라우드에 생성하는 것.
데이터는 안 넘어감. prisma db push로 빈 테이블만 만들어짐.
실제 유저 데이터는 배포 후 새로 쌓임.

----------------------------------------

[7] 유저: 테이블에 각자 데이터만 쌓이는 거야? 나는 모든 유저 데이터 관리 가능?

Claude:
- 유저: API에서 userId로 필터링하므로 자기 데이터만 보임
- 관리자(나): DATABASE_URL 가지고 있으므로 모든 데이터 조회 가능
- 관리 도구: Prisma Studio, Supabase 대시보드, pgAdmin 등

----------------------------------------

[8] 유저: PostgreSQL 쓰다가 클라우드 DB에 올리고 싶으면?

Claude: Neon 추천 (무료 512MB, Vercel 통합 좋음).
전환 순서: Neon 가입 → URL 복사 → .env.local 교체 → prisma db push → 끝.
로컬 데이터 옮기려면 pg_dump/psql로 백업/복원 가능.

----------------------------------------

[9] 유저: Vercel이랑 Neon 차이? 안드로이드/iOS 둘 다 하고 싶어

Claude:
- Vercel = 웹사이트/서버 호스팅
- Neon = 데이터베이스 호스팅
모바일 앱 방법: PWA, React Native(Expo), Capacitor
Expo 추천 - 맥 없이 iOS 빌드 가능, 코드 하나로 양쪽 다 됨.

----------------------------------------

[10] 유저: PWA는 안드로이드만? iOS는 맥북 필요?

Claude: PWA는 Android, iOS 둘 다 됨 (Safari에서 홈화면 추가).
앱스토어 등록하려면 Expo 쓰면 맥 없이도 클라우드 빌드로 가능.

----------------------------------------

[11] 유저: 앱스토어 등록 없이 어떻게 앱처럼 써?

Claude: PWA는 브라우저에서 바로 설치.
주소창 없이 전체화면으로 열리고, 앱 아이콘도 생김.
단점: 앱스토어 검색 불가, URL 직접 알려줘야 함.

----------------------------------------

[12] 유저: 앱스토어에도 내고 싶어

Claude: Expo(React Native) 사용.
백엔드 API는 그대로, 화면만 새로 만들면 됨.
웹+앱 둘 다 할 건지 확인 요청.

----------------------------------------

[13] 유저: 웹+앱 둘 다, iOS 전용으로 따로 만들어야 해?

Claude: Expo는 코드 1개로 Android + iOS 둘 다 빌드.
iOS 전용은 Swift/SwiftUI (맥 필수).
Expo가 압도적 유리: TypeScript 그대로, 맥 불필요, 코드 1개.

----------------------------------------

[14] 유저: 웹 백엔드 짜놓으면 모바일은 UI만 새로 하면 되는 거지?

Claude: 맞음. 같은 API를 웹/Android/iOS 세 플랫폼이 호출.
지금은 웹 완성에 집중하는 게 맞음.

----------------------------------------

[15] 유저: API가 라우팅이야?

Claude:
- 라우팅 = 페이지 이동 (화면을 보여줌)
- API = 데이터 처리 (JSON을 보내줌)
유저가 /feed 접속(라우팅) → 페이지가 /api/news/feed 호출(API) → 데이터 표시

----------------------------------------

[16] 유저: API는 어디에 어떻게 만들어?

Claude: src/app/api/ 폴더에 route.ts 파일 생성 = API 생성.
폴더 경로 = API 주소. GET/POST 함수 export하면 됨.
프론트에서 fetch()로 호출.

----------------------------------------

[17] 유저: API가 뭔지 더 자세히, 물리적 통로는 없는 거야?

Claude: 물리적 통로 없음. 코드로 짠 약속.
"이 주소로 요청하면 이 함수가 실행된다"는 규칙.
식당 키오스크 비유: 메뉴판=API, 메뉴번호=주소, 주방=함수, 요리결과=응답.

----------------------------------------

[18] 유저: 그럼 백엔드는 API 짜는 거네?

Claude: 백엔드 = API(창구) + 로직(실제 일하는 코드).
API는 src/app/api/, 로직은 src/lib/ 폴더.

----------------------------------------

[19] 유저: 서버는 뭐야?

Claude: 서버 = 백엔드 코드를 실행시켜주는 컴퓨터.
개발 중: npm run dev → 내 PC가 서버.
배포 후: Vercel이 서버 (24시간 대신 실행).

----------------------------------------

[20] 유저: 백엔드 기술 스택 다시 알려줘

Claude: Next.js, NextAuth.js, Prisma, Anthropic SDK, Stripe, RSS Parser,
한국투자증권 API, Finnhub API, Zod, bcryptjs, PostgreSQL 정리.

----------------------------------------

[21] 유저: Next.js는 뭐야? Prisma는 왜 쓰는 거야?

Claude:
- Next.js: React에 라우팅, API, SEO, 배포를 더한 프레임워크. React가 엔진이면 Next.js는 완성된 자동차.
- Prisma: SQL 대신 TypeScript로 DB를 다루게 해주는 ORM. SQL 번역기.

----------------------------------------

[22] 유저: SQLD 따려고 SQL 공부하려는데 비효율적일까?

Claude: 오히려 좋음. Prisma 쓰더라도 SQL 알면 디버깅, 최적화, 면접에 유리.
SQLD 공부하면 DB 설계(테이블 관계, 인덱스, 정규화) 이해에 직접 도움.

----------------------------------------

[23] 유저: Supabase 안 쓰고 PostgreSQL 쓰면 직접 테이블 관리하고 코드 짜야 돼?

Claude: Prisma가 다 해줌. 달라지는 건 DB 설치뿐.
schema.prisma 정의 → prisma db push → 테이블 자동 생성.
Prisma Studio로 브라우저에서 데이터 관리 가능.

----------------------------------------

[24] 유저: Prisma는 무슨 언어 써?

실제 코드에서는 전부 TypeScript. 새 언어 배울 필요 없음.
